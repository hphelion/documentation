When you execute the deploy process for a system, a copy of the builder files is also stored on every Object Storage node in the /etc/swiftlm/builder_dir/region-<region-name> directory. You can restore the builder files as follows:
  1. Log into the first proxy server node.
    Note: To identify the first proxy node, see HPE Helion OpenStack 2.0: Identifying the First Swift Proxy Server.
  2. Copy the swift-rings-tarball.tar file from any other node:
    scp <username>@<nodename>:/etc/swiftlm/deploy_dir/swift-rings-tarball.tar /tmp
  3. Unpack the tarball to /etc/swiftlm/builder_dir:
    cd  /etc/swiftlm/builder_dir
sudo tar -xpf /tmp/swift-rings-tarball.tar

  4. Log into the Deployer and run the swift-deploy.yml playbook::
    cd ~/scratch/ansible/next/hos/ansible
ansible-playbook -i hosts/verb_hosts swift-deploy.yml
