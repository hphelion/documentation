HPE Helion OpenStack® 2.0: Deployment Fails With - MSDOS Disks Labels Do Not Support Partition Names

Note: This process only applies to Swift drives. It does not apply to the operating system or boot drive.
Description If a disk drive allocated to Swift uses the MBR partition table type, the deploy process refuses to label and format the drive. This is to prevent potential data loss. (For more information, see HPE Helion OpenStack 2.0: Allocating Disk Drives for Object Storage. If you intend to use the disk drive for Swift, you must convert the MBR partition table to GPT on the drive using /sbin/sgdisk.
Resolution You must install gdisk, before using sgdisk.
  • Run the following command to install gdisk:
    sudo apt-get install gdisk
  • Convert to the GPT partition type. Following is an example for converting /dev/sdd to the GPT partition type:
    sudo sgdisk –g /dev/sdd
  • Reboot the node to take effect. You may then resume the deployment (repeat the playbook that reported the error).
