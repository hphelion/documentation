The swift utility (or CLI) is installed on the deployer/lifecycle-manager node and also on all other nodes running the Swift proxy service. To use this utility, set the following environment variables:

Note: You do not need to be the root user.
OS_USERNAME=demo
OS_PASSWORD=<redacted>
OS_TENANT_NAME=demo
OS_AUTH_URL=http://<cluster-name>-vip-admin-KEY-API-mgmt:35357/v2.0
OS_CACERT=/etc/ssl/certs/ca-certificates.crt
You must enter the appropriate password for the demo user and select the correct endpoint for the OS_AUTH_URL value. You can then examine the following account data:
$ swift stat
        Account: AUTH_205804d000a242d385b8124188284998
     Containers: 0
        Objects: 0
          Bytes: 0
X-Put-Timestamp: 1442249536.31989
     Connection: keep-alive
    X-Timestamp: 1442249536.31989
     X-Trans-Id: tx5493faa15be44efeac2e6-0055f6fb3f
   Content-Type: text/plain; charset=utf-8
In the above example, there are no containers. Execute the following command and create a container:
swift post documents
The newly created container appears. But there are no objects:

Upload a document:
$ swift upload documents mydocument
mydocument
List objects in the container:
$ swift list documents
mydocument
