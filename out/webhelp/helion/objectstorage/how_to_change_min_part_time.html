The following procedure enables you to change the min-part-time attribute.

Description In the cloud model ring-specifications, the min-part-time attribute specifies how many hours you must wait between ring rebuilds. If you change this attribute, this is not reflected in the ring files and it only applies at the time the rings are created.
Resolution You can change the values as follows:
  1. Log into the first server running the swift-proxy service.
  2. Go to the /etc/swiftlm/builder_dir/region-<region-name> directory, where <region-name> is the region-name field in your cloud model (for example, region1 in most example files).
  3. Search for files with a .builder extension.
  4. Run the swift-ring-builder command against each builder file. For example, to change the value to 16 hours for the account builder file, use the following: 
    sudo swift-ring-builder account.builder set_min_part_hours 16
  5. You should also update the ring-specifications (usually in the data/swift/rings.yml file) to contain the same value for the min-part-time.

For more details about the min-part-power attribute, see HPE Helion OpenStack 2.0: Understanding Ring Specifications.
