This page displays the alarms for the Helion Development Platform services. By default, alarms are sorted by most recent and in a table format.

Search and Sort

The search bar allows you to filter the alarms you want to view. Type and Status are examples of alarm criteria that can be specified. Additionally, you can filter by typing in text similar to searching by keywords.

The checkbox allows you to select (or deselect) a group of alarms and apply a single condition on the selected alarms:

  • Select Visible allows you to set a condition on selected alarms from the table.
  • Select All allows you to set a condition on all the alarms from the table.
  • Clear Selection allows you to clear all the selections currently selected from the table.

You can display the alarms by grid, list or table views by selecting the corresponding icons next to the Sort By control.

The Sort By control contains a drop-down list of ways by which you can sort the alarms. Alternatively, you can also sort using the column headers in the table.

  • Sort by State displays the alarms by reported status of OK, Warning, Error, and Unknown
  • Sort by Alarm displays the alarms by name.
  • Sort by Alarm ID displays the alarms UUID.
  • Sort by Last Check displays the alarms by the date and timestamp of the most recent occurence.
  • Sort by Dimension displays the alarms by device, hostname and service which they belong to.
  • Sort by Condition displays the alarms by whether the alarm is Open, Acknowledged, or Resolved.

Alarm Table

Each row has a checkbox to allow you to select multiple alarms and set the same condition on them.

The State column displays a graphical indicator that shows the state of each alarm:

  • Green square (or green indicator): OK. Good operating state.
  • Yellow diamond (or yellow indicator): Warning. Low severity, not requiring immediate action.
  • Red ball (or red indicator): Alarm. Varying severity levels and must be addressed.
  • Gray square (or gray indicator): Undetermined.

The Alarm column identifies the alarm by the name it was given when it was originally created.

The Last Check column displays the date and time the most recent occurence of the alarm.

The Dimension column describes the components to check in order to clear the alarm.

The Condition column shows the current condition of each alarm.

The last column, depicted by three dots, reveals an Actions menu that allows you to choose:

  • View Details, which opens a separate window that shows all the information from the table view and the alarm history. This window also allows you set the condition on the alarm to another condition. Note, the condition the alarm is currently set to is not selectable, as indicated by the grayed-out button.
  • Set Condition, which opens a separate window that allows you to directly apply a condition (Open, Acknowledged, or Resolved) to that alarm, which can be used for tracking and sorting purposes.
