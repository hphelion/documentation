Ceph is an open source, scalable, software-defined storage (SDS) platform that provides block and object storage with unified management.

The HPE Helion OpenStack 2.0 Ceph Storage Solution provides an unified, scalable, and stable storage solution for the management of the HPE Helion OpenStack Volume Storage (Cinder persistent Volumes) service.

For more information about installation, configuration, and integration between HPE Helion OpenStack: 2.0 and Ceph Firefly 0.80.7 running on the hlinux kernel 3.14.44-1, see HPE Helion OpenStack 2.0: Configuring for Ceph Block Storage Backend.

HPE Helion OpenStack introduces the concept of cloud models, where the user can modify the YAML to provide the required configuration based on their environment. To deploy Ceph, and make configuration changes, you must modify the cloud input model files related to Ceph.

You can implementing the input model and perform the following Ceph activities:
  • HPE Helion OpenStack 2.0: Configuring for Ceph Block Storage Backend
  • HPE Helion OpenStack 2.0: Adding an Object Storage Daemon (OSD) Node
  • HPE Helion OpenStack 2.0: Adding a New Ceph Monitor Node
  • HPE Helion OpenStack 2.0: Deploying Ceph Monitor Services on Dedicated Resource Nodes
  • HPE Helion OpenStack 2.0: Defining a Ceph Monitor Resource
  • HPE Helion OpenStack 2.0: Reconfiguring Ceph Services
  • HPE Helion OpenStack 2.0: Removing Object Storage Daemon (OSD) Node
  • HPE Helion OpenStack 2.0: Removing Ceph Monitor Node
  • HPE Helion OpenStack 2.0: Starting, Stopping, and Status for Ceph
Note: Ceph upgrade is a built-in, integral part of the HPE Helion Lifecycle Management upgrade, and is in line with rest of the supported services. In HPE Helion OpenStack 2.0, the supported version of Ceph is Firefly 0.80.7. In upcoming HPE Helion OpenStack releases, new versions of Ceph will be supported and the cluster can be upgraded to new releases. You should not use other releases of Ceph (for example, Firefly 0.80.10) and run Ceph upgrade verbs to upgrade the cluster. Doing this might cause a cluster stability issue.
