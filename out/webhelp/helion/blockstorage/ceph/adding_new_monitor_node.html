In HPE Helion OpenStack 2.0, a Ceph monitor node is installed on the Controller nodes by default. A new monitor node cannot be added to an existing Ceph cluster, if Ceph is deployed with the default configuration. However, if monitor nodes are added on a dedicated resource node, then you can add a new monitor node.

If a monitor resource is not already defined, you will need to first define one and then add the new monitor node. For instructions, see HPE Helion OpenStack 2.0: Defining a Ceph Monitor Resource.

To Add a New Monitor Node

Perform the following procedure to add a new monitor node to an existing Ceph cluster:
  1. Login to the Deployer/lifecycle-manager node.
  2. Edit the ~/helion/my_cloud/definition/data/servers.yml file to define a new monitor node as shown below:
    - id: mon4
  ip-addr: 192.168.10.11
  role: MON-ROLE
  server-group: RACK3
  nic-mapping: MY-2PORT-SERVER
  mac-addr: 8b:f6:9e:ca:3b:7a
  ilo-ip: 192.168.9.11
  ilo-password: password
  ilo-user: admin
    Note: Substitute the values in above sample as per your environment for the server being added.
  3. Commit your configuration: 
    cd ~/helion/hos/ansible
git add -A
git commit -m "Added monitor node {node-id}"
    Note: Replace the {node-id} in the commit message with appropriate value (for example, mon4).
  4. Run the following command:
    cd ~/helion/hos/ansible/
ansible-playbook -i hosts/localhost cobbler-deploy.yml
  5. To reimage all the nodes using PXE, run the following command:
    ansible-playbook -i hosts/localhost bm-reimage.yml
  6. Run the configuration processor:
    ansible-playbook -i hosts/localhost config-processor-run.yml
  7. Use the playbook below to create a deployment directory: 
    ansible-playbook -i hosts/localhost ready-deployment.yml
  8. Run the following commands: 
    cd ~/scratch/ansible/next/hos/ansible
ansible-playbook -i hosts/verb_hosts guard-deployment.yml
ansible-playbook -i hosts/verb_hosts osconfig-run.yml
ansible-playbook -i hosts/verb_hosts site.yml
  9. If you want the new compute node to be included in the monitoring service checks, there is an additional playbook that must be run to ensure this happens: 
    cd ~/scratch/ansible/next/hos/ansible
ansible-playbook -i hosts/verb_hosts monasca-deploy.yml --tags active_ping_checks

The new monitor node is deployed in an existing Ceph cluster.

To Verify the Newly-Added Monitor Node

From the Deployer/lifecycle-manager node, run the following command:
ceph --cluster ceph -f json quorum_status
Note: If the Ceph cluster name is different than the default (i.e. ceph), you need to modify the above command accordingly.

The newly-added monitor's host name appears in the quorum_names section of the output of above command.
