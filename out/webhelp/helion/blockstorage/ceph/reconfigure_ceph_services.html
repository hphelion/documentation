HPE Helion OpenStackÂ® 2.0: Reconfiguring Ceph Services

Perform the following steps to reconfigure the Ceph services.
  1. Login to the Deployer/lifecycle-manager node
  2. Configure the following parameters in the ~/helion/my_cloud/config/ceph/settings.yml file.
    • fsid
    • ceph clusters
    • osd_settle_time
    • OSD_journal_size
  3. Any additional configuration parameters for Ceph (beyond those listed in the ~/helion/my_cloud/config/ceph/settings.yml file) can be specified in the same file under the extra: category:
    extra:
  osd:
    journal max write entries: 200
    As shown above, configures the OSDs to write a maximum of 200 entries (journal max write entries: 200) to the journal at any one time.
    Note: The Ceph log parameter debug auth configures the verbosity of the authentication related logs, which is default configured to level of '1/5'. It is a security risk and therefore highly recommended not to have the 'debug auth' logging verbosely, as, the module will start logging the Ceph user secret's, which will be available to users via the centralized logs. Making Ceph cluster vulnerable.
  4. Commit your configuration:
    cd ~/helion/hos/ansible
git add -A
git commit -m "<your commit message>"
  5. Use the playbook below to create a deployment directory: 
    ansible-playbook -i hosts/localhost ready-deployment.yml
  6. Run the configuration processor: 
    cd ~/helion/hos/ansible
ansible-playbook -i hosts/localhost config-processor-run.yml
  7. Run the playbook to reconfigure the Ceph services:
    $cd ~/scratch/ansible/next/hos/ansible
$ansible-playbook -i hosts/verb_hosts ceph-reconfigure.yml
