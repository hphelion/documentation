HPE Helion OpenStackÂ® 2.0: Removing Object Storage Daemon (OSD) Node

Perform the following steps to remove the object storage daemon (OSD) node:
  1. Login to the Deployer/lifecycle-manager node.
  2. Make a note of the OSD numbers on the OSD node to be removed. A sample of OSD node is as follows:
    $ ceph osd tree
# id    weight  type name       up/down reweight
-1      6       root default
-2      3               host padawan-ceph-ccp-ceph0003-mgmt
0       1                       osd.0   up      1
3       1                       osd.3   up      1
6       1                       osd.6   up      1
-4      3               host padawan-ceph-ccp-ceph0001-mgmt
1       1                       osd.1   up      1
4       1                       osd.4   up      1
7       1                       osd.7   up      1
-3      3               host padawan-ceph-ccp-ceph0002-mgmt
2       1                       osd.2   up      1
5       1                       osd.5   up      1
8       1                       osd.8   up      1

    For example, to remove the OSD node padawan-ceph-ccp-ceph0001-mgmt, the OSD numbers 1, 4, and 7 are relevant.

  3. Run the following command to stop the OSDs running on the specific host:
    cd ~/scratch/ansible/next/hos/ansible
ansible-playbook -i hosts/verb_hosts ceph-stop.yml --limit <osd-node-to-remove>
    Note: Replace the string (osd-node-to-remove) in above command with the OSD host to be removed from the cluster (for example, stratushelion-ccp-ceph0001-mgmt).
  4. Run the following command for each OSD number (obtained from step #2 above):
    ceph osd out <osd-number> --cluster <ceph_cluster>
ceph auth del osd.<osd-number> --cluster <ceph_cluster>
ceph osd crush remove osd.<osd-number> --cluster <ceph_cluster> 
ceph osd rm <osd-number> --cluster <ceph_cluster>
    Note: Replace the string <ceph_cluster> with the name of your Ceph cluster (default cluster name is ceph).
  5. To remove the OSD host from the Ceph CRUSH map, run the following command:
    ceph osd crush remove <host-name>
    Note: Replace the string <host-name> in the above command.
  6. Remove the section for the OSD host from the ~/helion/my_cloud/definition/data/servers.yml file.
  7. Commit your configuration:
    cd ~/helion/hos/ansible
git add -A
git commit -m "Remove OSD node <host-name>"
  8. Run the configuration processor:
    ansible-playbook -i hosts/localhost config-processor-run.yml
  9. Use the playbook below to create a deployment directory: 
    ansible-playbook -i hosts/localhost ready-deployment.yml
    Note: Replace the string <host-name> in the above commit message with the actual host name (e.g., padawan-ceph-ccp-ceph0001-mgmt as mentioned in the above example).
  10. Log in to the OSD node and un-mount the data and journal disks. Also, remove the mount instructions for the same disks from the /etc/fstab file.
